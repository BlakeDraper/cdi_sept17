Short-Term Network (STN)

What is it?

  • a system for collection, management and dissemination of USGS flood data

  • a network of sites where sensors are deployed

FEV is the public data viewing tool for the STN database

  • STN is a system (database and suite of products) that collects, manages and disseminates data during and after hazardous flooding events
  • also, it is literally a network of sites where sensors are deployed
  • FEV is the Flood Event Viewer and it is the public data viewing tool for the STN database

Short-Term Network (STN)

What's with the name?

  • short-term, temporary data sensors

  • supplements USGS long-term station network

  • The USGS Streamgages, National Groundwater Well Network and the Water Quality Network form a set of fixed, permanent stations where USGS collects water data
  • STN supports dense, short-term data collection from temporary sensors
  • The Short-Term Network of sites is deployed in between our Long-Term station network, to supplement data needs for large flooding events.

Short-Term Network (STN)

Why is it needed?

  • standardize data collection process

  • new repository for historical high-water marks

...has evolved into a suite of products

  • STN began as a pilot program to support the flood data collection efforts of the Water Science Centers - there was a need for a tool to make this process easier and more standard
  • Another goal was to provide a long-term repository for historical high-water marks which would be accessible to outside partners such as ASFPM (among many others)
  • This pilot program has gained momentum and evolved into an indispensable suite of products that promote good data collection practices, data management and dissemination to partners and the public

Event-Centrism in FEV

all data gets associated with an event

Water Science Centers and the press want event-focused map

  • The STN universe promotes the event as the central organizing concept. All sensor deployments, peak summaries and HWMs are associated with an event.
  • Both the Water Science Centers and the press generally want an event-focused map during an event
  • STN is designed to be event-centric from the bottom up.

High-water Mark layer

observed & surveyed

historic HWM effort underway

  • HWMs are observed and elevation is surveyed after waters have receded

  • Historic high-water marks from events dating back as far as 1888

Interpreted Data

    Peak Summary

Peak Summary layer

calculated after the event

interpreted data - based on multiple input

  • Peak summaries are calculated after the event and provide the peak water level for that event at that site. Generally found where there were storm tide sensors deployed
  • The peaks are the interpreted final dataset that illustrate the maximum water elevation for the chosen event - a combination of several pieces of data
